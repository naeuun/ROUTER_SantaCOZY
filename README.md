# ROUTER_SantaCOZY
2025 디지털소프트웨어공학부 학생회 ROUTER 연말 이벤트 - 산타 코지

## 1️⃣ 브랜치 규칙

| 브랜치 이름                  | 역할              |
| ----------------------- | --------------- |
| `main`                  | 최종 배포용 브랜치      |
| `BACKEND`               | 백엔드 공용 작업 브랜치   |
| `FRONTEND`              | 프론트엔드 공용 작업 브랜치 |
| `fe/feature/기능명` | 프론트엔드 기능 개발 브랜치 |
| `be/feature/기능명` | 백엔드 기능 개발 브랜치   |
| `fe/fix/세부사항`     | 프론트엔드 버그 수정 브랜치 |
| `be/fix/세부사항`     | 백엔드 버그 수정 브랜치   |

### 📌 브랜치 이름 예시

```
fe/feature/login-page
be/feature/login
fe/fix/button-color
```

---

## 2️⃣ 커밋 메시지 규칙

```
<타입>: <작업 내용> 
```

### 📌 커밋 타입

| 타입       | 의미         | 예시                              |
| -------- | ---------- | ------------------------------- |
| ✨feat     | 새로운 기능 추가  | feat: 회원가입 페이지 UI 구현 |
| 🔧fix      | 버그 수정      | fix: 로그인 API 에러 수정    |
| 📄docs     | 문서 수정      | docs: README 업데이트         |
| ♻️refactor | 코드 리팩토링    | refactor: API 응답 구조 개선|
| ⚙️chore    | 빌드/설정 변경 등 | chore: eslint 설정 추가   |

- ex) ✨feat: login 기능 추가

---

## 3️⃣ 이슈 작성 규칙

### 제목 예시

```
✨ Feature - 회원가입 화면 구현
🐛 Bug - 로그인 오류 발생
```

### 내용 작성 예시

* **기능 추가(Feature)**

  ```
  ## 📌 이슈 내용
  - 어떤 기능을 추가할지 구체적으로 작성해주세요.

  ## 📋 상세 내용 (선택)
  - 필요한 세부 내용 작성

  ## ✅ 완료 조건
  - [ ] 기능이 정상 동작함
  - [ ] 문서 업데이트 완료
  ```

* **버그 리포트(Bug)**

  ```
  ## 🔎 버그 설명
  - 어떤 버그인지 자세히 적어주세요.

  ## 📌 재현 방법 (선택)
  1. 어떤 상황에서 버그가 발생하는지 단계별로 작성
  2.

  ## ✅ 예상 동작
  - 버그가 없을 때 정상적으로 작동하는 모습
  ```

## 4️⃣ PR(Pull Request) 작성 규칙

### PR 제목

```
[✨ Feature][FE] 회원가입 화면 구현
[🐛 Bug][BE] 로그인 오류 수정
```

### PR 본문 템플릿

```
## #️⃣ 연관된 이슈
- #12
- #15

## 📢 작업 내용 
> 이번 PR에서 작업한 내용을 간략히 설명해주세요. (이미지 첨부 가능)
- (프론트엔드) 관련 와이어프레임 또는 프로토타입 이미지
- (백엔드) 관련 기능 명세서 링크 or 간략히 설명 

## ✨ Changes
- 어떤 부분이 어떻게 바뀌었는지 작성해 주세요.
- 기능을 어떻게 구현했는지 구체적으로 설명해 주세요.

## 📷 스크린샷 (선택)
- 스크린샷, 시연 영상

## 💬 리뷰 요구사항 (선택)
- 집중적으로 리뷰를 원하는 부분 작성
  ex) 메서드 명이 적절한지 검토 부탁드립니다.
```

## 5️⃣ 작업 기본 흐름

1. **이슈 생성** → 기능/버그에 맞는 템플릿으로 작성
2. **브랜치 생성**

   ```bash
   git checkout BACKEND   # 또는 FRONTEND
   git pull origin BACKEND
   git checkout -b be/feature/기능명
   ```
3. **작업 & 커밋**

   ```bash
   git add .
   git commit -m "feat: 로그인 API 추가 "
   git push origin be/feature/기능명
   ```
4. **PR 생성 및 리뷰** → `BACKEND` 또는 `FRONTEND` 브랜치로 PR
5. **코드 리뷰 후 병합(Merge)**
6. **최종 배포 전 main 브랜치에 병합**
